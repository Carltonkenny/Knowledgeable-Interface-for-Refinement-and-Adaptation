What PromptForge Is
Not a prompt improver. A personal prompt intelligence layer that compounds value over time.

The Core Problem

Non-technical users get poor results from AI tools because their prompts are vague, contextless, and unstructured. Every other prompt tool takes input and returns output — silently, generically, with no memory of who you are.

PromptForge does something fundamentally different. It learns how you think, what you build, and how you communicate. The more you use it, the better it serves you. Switching away means starting over — that is the moat.

The Data Flywheel
More usage → richer profile → smarter routing → better prompts → more usage. This compounds. A competitor can copy the architecture. They cannot copy your users' accumulated history.

What Makes It Different

  • Kira — an orchestrator with a distinct personality that adapts to each user
  • 4-agent parallel swarm with selective firing based on what is actually needed
  • Two memory layers — structured signals in Supabase, semantic context in LangMem
  • Prompt diff and quality scoring — shows users what changed and why
  • Multimodal — text, files, images, voice all feed the same pipeline
  • MCP server — works inside Cursor, Claude Desktop, any MCP-compatible tool
  • Profile seeded on onboarding — never starts cold
  • Progress visibility — users see their prompt quality improving over time

Kira — The Orchestrator Personality
Not a router. A personality with routing capability. The face of the product.

Character Definition

Kira is direct, warm, and slightly opinionated. She speaks like a senior collaborator who respects your time. She never uses filler phrases. She never asks more than one question at a time. She pushes back when your prompt needs it.

CONSTANTS — NEVER CHANGE
  • ✗ Never says "Certainly", "Great question", "Of course"
  • ✗ Never asks more than one question
  • ✗ Never explains her process in detail
  • ✓ Always reads profile before speaking
  • ✓ Always has an opinion on your prompt
  • ✓ Speed is a personality trait — she moves fast

Name Rationale

Kira — sharp, decisive, gender-neutral feeling. Sounds like someone who moves fast and knows what they're doing. The K sound gives confidence without aggression. Memorable without being obscure.

Alternatives if reconsidering: Prism (concept maps to multi-angle analysis), Vesper (premium, distinctive), Lumen (clarity and illumination).

Two-Layer Memory System
LangMem and Supermemory serve different surfaces and never conflict.

LangMem — Internal Pipeline Memory

OWNS THE APP SURFACE

Lives inside LangGraph pipeline. Native integration — no adapter layer. Stores operational intelligence that feeds Kira's routing decisions and agent analysis.

  • Prompt quality history and trends
  • Domain confidence per user
  • Past improved prompts as stylistic reference
  • Clarification patterns and outcomes
  • Agent skip decisions and their accuracy
  • Session-level project context
Query: message as search → returns relevant memories
Write: after each session as background task
Latency: native to LangGraph · ~50ms

Supermemory — MCP Surface Memory

OWNS THE MCP SURFACE

Handles context injection when user is in Cursor, Claude Desktop, or any MCP tool. Not called during normal web app requests.

  • Conversational facts mentioned in passing
  • Project context and client details
  • Soft preferences stated naturally
  • Temporal updates — "I moved to SF" supersedes "I live in NYC"
  • MCP context prompt injection at conversation start
Called: MCP requests only
Write: lighter summary after each session
Latency: ~50ms per query
The Rule — They Never Conflict
LangMem runs on web app requests. Supermemory runs on MCP requests. They write different content — operational signals vs conversational context. They never read each other. Your core moat (LangMem operational intelligence) lives in your own infrastructure, not in a third-party system.

Frontend — Interface Design
Professional and clean. Everything in the chat input. No separate upload zones.

Input Bar — Exactly Like Your Screenshot

Single input area at the bottom. Three icons to the right: plus for file/image, microphone for voice, send button. No visible modality switching. Everything in the conversation flow.

  • Plus icon → bottom sheet with two options: choose file or upload image. 5MB max for images, 2MB max for text files.
  • Microphone → tap to record, waveform animates, transcript populates input field for review before send.
  • Waveform icon → live audio visualizer during recording.
  • Attachment pill → small thumbnail or filename above input. Removable before send.

Three Visual Zones

  • User messages — right aligned, clean bubble
  • Kira's messages — distinct visual treatment, left aligned, Kira avatar
  • Improved prompt output — card with diff highlight, quality score, copy button

Personalization Indicator

Small colored dot in the input bar corner. Users learn over time what it means without explanation.

Grey dot
Cold — Kira doesn't know you yet
Amber dot
Warm — Kira has your patterns
Green dot
Tuned — Kira knows your work deeply

Quality Score Display

Below the improved prompt. Three thin horizontal bars. Specificity · Clarity · Actionability. Clean numbers 1-5. No graphs. No clutter.

Reply Threading

Exactly like your screenshot. User can reply directly to Kira's improved prompt output with "make it shorter" — the reply context tells the system exactly which output they're refining. No ambiguity in FOLLOWUP classification.
