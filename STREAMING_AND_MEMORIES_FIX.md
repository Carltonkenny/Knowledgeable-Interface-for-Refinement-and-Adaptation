## 🐛 ISSUE 2: NO TOKEN STREAMING

### **Problem:**
- Full response appeared at once (no typewriter effect)
- Console showed: `kira_message {message: "Full response here", complete: false}`

### **Root Cause:**
The handler used `llm.invoke()` which **blocks until complete**:

```python
# agents/autonomous.py line 720
response = llm.invoke([...])  # ← Blocks for 1-3s, returns FULL response

# api.py line 548
yield _sse("kira_message", {"message": reply, "complete": False})  # ← Sends ALL at once
```

### **Fix Applied:**

#### **Step 1: Created Streaming Handler**

**File:** `agents/autonomous.py` line 748-801

**Added NEW function:**
```python
async def kira_unified_handler_stream(
    message: str,
    history: list,
    user_profile: dict
):
    """
    Streaming version of kira_unified_handler.
    Yields tokens as they're generated by the LLM.
    """
    try:
        llm = get_fast_llm()
        context = build_kira_context(message, history, user_profile)
        
        # Stream tokens as generated ← KEY DIFFERENCE!
        async for chunk in llm.astream([
            SystemMessage(content=KIRA_UNIFIED_PROMPT),
            HumanMessage(content=context)
        ]):
            if chunk.content:
                yield chunk.content  # ← Yield each token!
    
    except Exception as e:
        logger.error(f"[kira_unified_stream] failed: {e}")
        yield "I encountered an error. Please try again."
```

#### **Step 2: Updated API to Stream**

**File:** `api.py` line 547-562

**Added:**

# Stream Kira's response token-by-token for CONVERSATION/FOLLOWUP
if intent in ["CONVERSATION", "FOLLOWUP"]:

## ⚠️ LIMITATION: NEW_PROMPT MODE

### **Current Behavior:**

For **NEW_PROMPT** intents (e.g., "Write a Python function"), the system uses the **swarm pipeline** (4 agents), not the unified handler.

**Why no streaming for NEW_PROMPT?**

The swarm runs **4 parallel LLM calls**:
```
Intent Agent → Context Agent → Domain Agent → Prompt Engineer
     ↓              ↓              ↓              ↓
  (parallel execution, then join)
```

This is **inherently batch** - can't stream until all agents complete.

### **Workaround:**

Status updates still stream progressively:
```
0ms:    "Analyzing intent..."
200ms:  "Extracting context..."
400ms:  "Identifying domain..."
600ms:  "Engineering your prompt..."
        ↓
2000ms: [Full response appears]
```

### **Future Enhancement:**

To stream NEW_PROMPT responses, would need to:
1. Refactor swarm to stream from Prompt Engineer
2. More complex (requires LangGraph changes)
3. **Priority:** 🟡 LOW - CONVERSATION/FOLLOWUP are more common
